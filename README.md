# Chiara's Kitchen

This repo describes how I work: how I set goals, plan my day, manage tasks and projects, take notes, and more. It uses the metaphor of a kitchen, specifically:

- I have a cookbook — a collection of recipes that work for me.
- I have tools, and I think carefully before adding new ones.
- I have a test kitchen — I don't just add new recipes on a whim. I make proposals, clarify the problem I'm trying to solve, explore ideas, and test them before adding anything to the cookbook.

## Why this exists

Two things that prompted it.

**Changing things for the wrong reasons.** I have TickTick, it works well for me, and I was thinking about switching to Google Tasks because it lives beside Google Calendar — a nice-to-have, not a real problem. Without a process, the system ends up being whatever I last read about.

**Not knowing how to fix what isn't working.** The temptation is to overhaul everything. But smaller, deliberate changes work better: define the problem, try something specific, see if it helped.

## Influences

- **Cal Newport** — rooted productivity: a system that's stable and deliberate rather than constantly reinvented. This kitchen is an iterative version of that.
- **BJ Fogg, Tiny Habits** — start with the smallest viable version of a behavior, not the full thing.

## How new recipes get added

**Discovery**

Every time something comes to mind — a problem, an idea, something I want to change — I create a GitHub issue with two things: the idea, and the problem it's trying to solve. The problem comes first.

Before trying anything, I explore options with Claude, with a bias toward solving it within the system I already have rather than adding something new.

**Delivery**

Once there's something worth trying, it goes into the test kitchen in two stages:

1. **10-day trial** — is this viable? Does it feel like it could work?
2. **30-day challenge** — does it actually solve the problem, and can I sustain it?

If it passes both, it earns a place in the cookbook.

## What's in here

- **Recipes** — active, tested practices
- **Tools** — tools I use and think carefully about adding
- **Test Kitchen** — current trials
- **Archive** — retired recipes, with notes on what broke
- **Issues** — proposals and ideas in progress → [GitHub Issues](../../issues)

### Issue statuses

| Status | When to use |
|--------|-------------|
| **Inbox** | Idea captured with the problem it's trying to solve |
| **Exploring** | Investigating options |
| **In Trial** | Active 10-day or 30-day test kitchen phase |
| **Done** | Added to cookbook or closed without adoption |
